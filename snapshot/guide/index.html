Micronaut Ignite

Integration between Micronaut and Apache Ignite

Version:

1 Introduction

Apache Ignite is a horizontally distributed fault tolerant compute platform for real-time applications that can process terabytes of data with in-memory speed.

This project adds support for Apache Ignite and bridges support to other Micronaut components : Micronaut Cache

1.1 Release History

1.0.0

  • Micronaut Ignite

2 Ignite Quick Start

To add support for Ignite to an existing project, you should first add the Micronaut Ignite Core configuration to your build configuration. For example in Gradle:

implementation("io.micronaut.ignite:micronaut-core-ignite")
<dependency>
    <groupId>io.micronaut.ignite</groupId>
    <artifactId>micronaut-core-ignite</artifactId>
</dependency>

The minimum requirement for configure Ignite is set the value of the ignite.enabled property in application.yaml.

2.1 Ignite Client Configuration

Configuration Ignite
ignite:
  enabled: true
  clients:
    default:
      path: "classpath:default.cfg"

A config file is needed to setup ignite and this configured through a spring bean xml configuration. Additional configurations can be found here: https://github.com/apache/ignite/blob/master/examples/config.

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xsi:schemaLocation="
        http://www.springframework.org/schema/beans
        http://www.springframework.org/schema/beans/spring-beans.xsd">
    <bean id="basic.cfg" class="org.apache.ignite.configuration.IgniteConfiguration">
        <property name="clientMode" value="true"/>
        <property name="discoverySpi">
            <bean class="org.apache.ignite.spi.discovery.tcp.TcpDiscoverySpi">
                <property name="ipFinder">
                    <bean class="org.apache.ignite.spi.discovery.tcp.ipfinder.multicast.TcpDiscoveryMulticastIpFinder">
                        <property name="addresses">
                            <list>
                                <value>localhost:47500..47509</value>
                            </list>
                        </property>
                    </bean>
                </property>
            </bean>
        </property>
    </bean>
</beans>

The full list of configurable options via micronaut properties is below:

path to load in bean configuration.

2.2 Ignite Thin Client Configuration

Ignite supports a thin-client configuration that does not become apart of the topology. A minimum thin client configuration is provided below:

Configuring Ignite Thin Client
ignite:
  enabled: true
  thin-clients:
    default:
      address:
      - "127.0.0.1:10800"
      - "127.0.0.1:10801"

3 Micronaut Cache

To take advantage of Micronaut Cache, this can be enabled by including the following dependency:

implementation("io.micronaut.ignite:micronaut-cache-ignite")
<dependency>
    <groupId>io.micronaut.ignite</groupId>
    <artifactId>micronaut-cache-ignite</artifactId>
</dependency>

Note: Information usage in regards to micronaut cache can be found in the micronaut cache documentation.
